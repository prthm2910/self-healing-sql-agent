  <name>Implement SQLTranspiler Service</name>
  <files>src/services/sql_engine.py</files>
  <behavior>
    - Given a dictionary of snippets like `{'t1': 'SELECT film_id FROM film', 't2': 'SELECT film_id, customer_id FROM rental'}` and a join plan, it should produce a single SQL string using CTEs.
    - It must use SQLGlot to parse snippets and construct the final AST.
    - It should handle INNER joins between snippets based on the plan.
  </behavior>
  <action>
    - Add `SQLTranspiler` class to `src/services/sql_engine.py`.
    - Implement `merge_snippets(snippets: Dict[str, str], join_plan: Dict[str, Any]) -> str`:
      1. Start with `sqlglot.select("*").from_(join_plan["base_task"])`.
      2. Iterate through `join_plan["steps"]` to add `.join(...)` calls.
      3. Use `.with_(alias, as_=parse_one(sql))` for each snippet to wrap them in CTEs.
      4. Return the generated SQL string using `.sql(dialect="postgres")`.
    - This implements DAC-02 by using deterministic AST merging instead of LLM-based join generation (per D-DAC-03).
  </action>
  <verify>
    <automated>python -c "import sqlglot; from src.services.sql_engine import SQLTranspiler; print('SQLTranspiler implementation check')"</automated>
  </verify>
